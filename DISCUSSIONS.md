# Discussions (placeholder)

Until maintainer enables GitHub Discussions in repo settings → General,
this file documents intended category structure so the README/sidebar can
link here.

| Category | Purpose |
|---|---|
| **Announcements** | Schema version bumps (rounds v0.5.0 → v0.6.0), validator updates, profile retirements. |
| **General** | Open-ended chat, contributor introductions. |
| **Profiles** | "Why no profile for X?" / "How is profile Y structured?" / suggestion proposals. |
| **Schemas** | `ROUNDS_SCHEMA.md` / `AGENT_GUIDELINES_SCHEMA.md` discussion, schema-bump RFCs. |
| **PR feedback & rounds** | Drop links to a real PR (honcho / qdrant / etc.) and the rounds table others might find useful. |
| **Lessons (misakanet-50)** | Lesson submissions, scoring debate, cross-pollination to/from MisakaNet. |

## Why a placeholder file

pr-genius is content-first. Most of the metadata a community would post in
Discussions is already encoded in `index.md`, `KNOWN_ISSUES.md`, and
`<repo>/index.md` profiles. Discussions adds nothing that an indexed `md`
cannot — but it lets casual contributors talk without a PR.

## Enable

To turn on, repo admin → Settings → General → "Discussions" → enable
with the categories above.
